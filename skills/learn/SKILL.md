# /learn - 会话模式提取与学习

在会话中解决了非平凡问题后，手动调用此 skill 提取可复用模式并持久化。

## 触发方式

在会话任意时刻运行 `/learn`。

## 核心流程

### 第 1 步：会话分析

回顾当前会话，识别以下 5 类模式：

| 类型 | 说明 | 示例 |
|------|------|------|
| 错误解决方案 | 反复尝试后成功的修复 | 某框架版本兼容性问题的解决 |
| 用户纠正 | 用户纠正了 AI 的错误假设 | "不要用 class 组件，用 hooks" |
| 调试技巧 | 非显而易见的调试方法 | 某工具组合有效定位问题 |
| 工作流偏好 | 用户偏好的工具/流程 | "先 Grep 再 Edit"、"总是先跑测试" |
| 项目特定约定 | 项目独有的规范/模式 | 目录结构约定、命名规范、架构模式 |

**规则**：
- 忽略琐碎修复（拼写错误、简单语法错误）
- 忽略一次性问题（API 临时故障等）
- 聚焦可复用、能节省未来时间的模式
- 每个模式保持独立，一个模式一份文件

### 第 2 步：质量评估

对每个提取的模式，按 5 个维度打分（1-5 分）：

| 维度 | 1 分 | 3 分 | 5 分 |
|------|------|------|------|
| 具体性 | 仅有抽象原则，无代码示例 | 有代表性代码示例 | 覆盖所有使用场景的丰富示例 |
| 可操作性 | 不清楚该怎么做 | 主要步骤可理解 | 立即可执行，边界情况已覆盖 |
| 范围适配 | 范围过宽或过窄 | 基本适当，边界略有模糊 | 名称、触发条件、内容完美对齐 |
| 非冗余性 | 与已有 memory/skill 几乎重复 | 有重叠但有独特视角 | 完全独特的价值 |
| 覆盖度 | 仅覆盖目标任务的一小部分 | 主要场景已覆盖，常见变体缺失 | 主要场景、边界情况、陷阱全覆盖 |

**质量门槛**：所有维度 >= 3 分才保存。如有维度 < 3，改进草稿后重新评分。

### 第 3 步：作用域决策

问自己："这个模式在其他项目中也有用吗？"

| 作用域 | 判断标准 | 保存位置 |
|--------|----------|----------|
| 项目级 | 项目特定知识（代码风格、框架约定、架构决策） | `~/.claude/projects/{project}/memory/MEMORY.md` |
| 全局级 | 通用模式（调试技巧、最佳实践、工具使用） | `~/.claude/skills/learned/{pattern-name}.md` |

**原则**：拿不准时选全局（从全局迁到项目比反过来容易）。

### 第 4 步：草稿输出

#### 项目级模式 → 追加到 MEMORY.md

直接追加到对应项目的 `MEMORY.md`，格式自然融入现有结构。

#### 全局级模式 → 新建 skill 文件

保存到 `~/.claude/skills/learned/{pattern-name}.md`：

```markdown
---
name: pattern-name
description: "简短描述（130 字以内）"
user-invocable: false
origin: auto-extracted
---

# 模式名称

**提取时间：** YYYY-MM-DD
**适用场景：** 简要描述适用条件

## 问题
具体描述这个模式解决什么问题

## 解决方案
模式/技巧/变通方案（含代码示例）

## 何时使用
触发条件列表
```

### 第 5 步：确认保存

向用户展示：
1. 评分表（5 维度分数 + 理由）
2. 保存路径
3. 最终草稿内容

**等待用户明确确认后再写入文件。**

## 输出格式

```
## 会话学习报告

### 发现的模式

1. **[模式名称]** (类型: 错误解决方案)
   - 简述: ...
   - 作用域: 项目级 / 全局级

### 质量评估

| 维度 | 分数 | 理由 |
|------|------|------|
| 具体性 | N/5 | ... |
| 可操作性 | N/5 | ... |
| 范围适配 | N/5 | ... |
| 非冗余性 | N/5 | ... |
| 覆盖度 | N/5 | ... |
| **总分** | **N/25** | |

### 保存位置

- 路径: `...`
- 状态: 待确认 ✋

确认保存？[等待用户回复]
```

## 与现有系统的关系

- **MEMORY.md**：项目级模式直接追加到项目 memory
- **~/.claude/skills/learned/**：全局模式保存为独立 skill 文件
- **/instinct**：`/learn` 是手动提取，`/instinct` 是自动观察，两者互补
